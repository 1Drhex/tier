Guest Question.

Production Scheduling at Toyisha Auto\(.^{5}\)
The Toyisha Auto company manufactures five car models on five different machines. The hourly requirements of the machines for the different models, as well as the number of cars of each model to produce, are listed in Table 8.4. For example, a car of model 1 requires an hour from machines 1, 2, and 3, and four model 1 cars must be produced. The total in each machine column is the total number of hours required, for all cars, on that machine. (It is a sumproduct of the machine column and the last column.) We assume that the machines are used simultaneously. For example, a car of model 1 takes one hour to produce, utilizing an hour of machine 1, machine 2, and machine 3 simultaneously. The company’s goal is to “smooth” machine utilization through time. For example, focus on machine 1. By the time all 25 cars have been produced, its total machine time will be 13 hours. The company would like 1/25 of this time to be utilized by the first car produced, 2/25 of this time to utilized by the first two cars produced, and so on—and similarly for the other machines. Of course, this smoothing can’t be achieved exactly, but Toyisha wants to find a production schedule that satisfies it as much as possible.

Table 8.4 Machine Requirements (Hours):

Cars Required Machine 5 Machine 4 Machine 3 Machine 2 Machine 1 Model
4 0 0 1 1 1 1
6 0 1 1 1 0 2
6 1 1 1 0 0 3
5 1 1 0 0 1 4
4 1 0 0 1 1 5
25 15 17 16 14 13 Total

Objective To use Evolutionary Solver to find the permutation of car production that smoothes machine utilization over time as much as possible.
Where Do the Numbers Come From?
All of the machine utilization numbers in Table 8.4 should be readily available from the company’s production processes, and the company knows how many cars of each model it wants to produce in the next time period.

Admin
Solution

No “big picture” is really needed for this problem. The idea is to index the cars from 1 to 25—1 to 4 for model 1 cars, 5 to 10 for model 2 cars, and so on—and then find the optimal permutation of these indexes. The completed model is shown in Figure 8.32 and the file Car Production Scheduled Finished.xlsx. It can be developed with the following steps.

1) Given data. Enter the data from Table 8.4 in the light blue cells. Note that the total in cell I10 is the sum of the numbers above it, and the totals in cells B10 to F10 are sumproducts of columns B to F and column I.
2) Car indexes. Enter any permutation of the integers 1 to 25 in the Car_index range. These are the decision variable cells.
3) Car model lookups. To find the model numbers from the car indexes, enter the formula
=VLOOKUP(B14,LTable1,2)
in cell C14 and copy it down column C.
4) Cumulative machine usage lookups. Each value in columns D to H from row 14 down is the cumulative number of hours utilized by that machine through that number of cars. You can calculate these with lookups. First, enter the formula
=VLOOKUP($C14,LTable2,D13+1,FALSE)
in cell D14 and copy it across to cell F14. Then enter the formula
=D14+VLOOKUP($C15,LTable2,D$13+1,FALSE)

and copy it down and across. Note that the bottom row, row 38, necessarily matches the totals in row 10.
5) Desired cumulative machine usages. The company wants the machine usages to increase linearly through the production process, so enter the formula
=($A14/$I$10)*B$10
in cell I14 and copy it to the range I14:M38.
6) Smoothing objective. One way to smooth the machine utilizations over time is to minimize the sum of absolute differences between the numbers in the range D14:H38 and those in the range I14:M38. (You could also minimize the sum of squared differences.) Calculate these absolute differences by entering the formula
=ABS(D14-I14)
in cell N14 and copying it to the range N14:R38. Then calculate the sum of these absolute differences with the SUM function in cell B40.

Using Evolutionary Solver
The Solver dialog box (not shown here) is almost identical to the one for the traveling salesperson model in the previous example (see Figure 8.30). There are no constraints other than the “alldifferent” constraint on the decision variable cells, and the objective is to minimize the sum of absolute differences in cell B40.

Discussion of the Solution
The optimal permutation shown in Figure 8.32 would be practically impossible to guess. It tries to get the actual cumulative machine usages as close to the desired linear cumulative usages as possible—and do this simultaneously for all machines. The graph in the figure shows how close it comes for machine 1, and similar graphs could be created for the other machines. You might try other permutations in the red cells to see how much worse (less smooth) the machine utilizations become. It’s not easy!

Other Uses for the AllDifferent Constraint
We introduced the alldifferent constraint in the context of traveling salesperson and production scheduling models, but it has many other applications. Specifically, it can be used in any model where the solution is defined by a permutation of integers from 1 to n. You are asked to explore some possibilities in the problems.
