Guest Question.

Let L be the line in \(R^{2}\) spanned by vector \( \left[\begin{array}{r}3 \\1\\\end{array}\right]\) . Let T be the linear transformation from \(R^{2}\) to \(R^{2}\) that projects any vector \(\vec{v}\) orthogonally onto line L, as shown in Figure 5. We can facilitate the study of T by introducing a coordinate system where L is one of the axes (say, the \(c_{1}\)-axis), with the \(c_{2}\)-axis perpendicular to L, as illustrated in Figure 6. If we use this coordinate system, then T transforms \( \left[\begin{array}{l}c_{1} \\c_{2}\end{array}\right] \text { into }\left[\begin{array}{c}c_{1} \\0\end{array}\right]\). In \(c_{1}\)–\(c_{2}\), coordinates, T is represented by the matrix \(B=\left[\begin{array}{ll}1 & 0 \\0 & 0\end{array}\right]\), since \(\left[\begin{array}{c}c_{1} \\0\end{array}\right]=\left[\begin{array}{ll}1 & 0 \\0 & 0\end{array}\right]\left[\begin{array}{l}c_{1} \\c_{2}\end{array}\right]\).

Solution

Let‚Äôs make these ideas more precise. We start by introducing a basis ùï≠ =(\(\vec{v}_{1}\),\(\vec{v}_{2}\)) of \(R^{2}\) with vector \(\vec{v}_{1}\)on line L and vector \(\vec{v}_{2}\) perpendicular to L, for example,

\(\vec{v}_{1}=\left[\begin{array}{l}3 \\1\end{array}\right] \quad \text { and } \quad \vec{v}_{2}=\left[\begin{array}{r}-1 \\3\end{array}\right]\).

\(\text { If } \vec{x}=\underbrace{c_{1} \vec{v}_{1}}_{\text {in } L}+\underbrace{c_{2} \vec{v}_{2}}_{\text {in } L^{\perp}} \text {, then } T(\vec{x})=\operatorname{proj}_{L}(\vec{x})=c_{1} \vec{v}_{1} \text {. Equivalently, if }\)

\([\vec{x}]_{\mathfrak{B}}=\left[\begin{array}{l}c_{1} \\c_{2}\end{array}\right], \quad \text { then } \quad[T(\vec{x})]_{\mathfrak{B}}=\left[\begin{array}{c}c_{1} \\0\end{array}\right]\):

see Figure 6.

The matrix \(\text { The matrix } B=\left[\begin{array}{ll}1 & 0 \\0 & 0\end{array}\right] \text { that transforms }\vec{x}]_{\mathfrak{B}}=\left[\begin{array}{l}c_{1} \\c_{2}\end{array}\right] \text { into }T(\vec{x})]_{\mathfrak{B}}=\left[\begin{array}{c}c_{1} \\0\end{array}\right]\) is called the ùï≠-matrix of T :

\([T(\vec{x})]_{\mathfrak{B}}=B[\vec{x}]_{\mathfrak{B}}\)

We can organize our work in a diagram as follows:

\(\vec{x}=\overbrace{c_{1} \vec{v}_{1}}^{\text {in }L}+\overbrace{c_{2} \vec{v}_{2}}^{\text {in } L^{\perp}} \underset{T}{\longrightarrow} \quad T(\vec{x})=c_{1} \vec{v}_{1}\)

\([\vec{x}]_{\mathfrak{B}}=\left[\begin{array}{l}c_{1} \\c_{2}\end{array}\right] \quad \underset{B=\left[\begin{array}{ll}1 & 0 \\0 & 0\end{array}\right]} {\longrightarrow} [T(\vec{x})]_{\mathfrak{B}}=\left[\begin{array}{c}c_{1} \\0\end{array}\right]\).

When setting up such a diagram, we begin in the top left by writing an arbitrary input vector \(\vec{x}\) as a linear combination of the vectors in the given basis ùï≠. In the top right we have T(\(\vec{x}\)), again written as a linear combination of the vectors of basis ùï≠. The corresponding entries below are the coordinates vectors \([\vec{x}]_{\mathfrak{B}} \text { and }[T(\vec{x})]_{\mathfrak{B}}\). Finding those is a routine step that requires no computational work, since \(\vec{x}\) and T(\(\vec{x}\)) have been written as linear combinations of the basis vectors already. Finally, we find the matrix B that transforms \([\vec{x}]_{\mathfrak{B}} \text { into }[T(\vec{x})]_{\mathfrak{B}}\); this is again a routine step.
