Guest Question.

Image-based homing for robots can be implemented by generating heading command inputs to a steering system based on the following guidance algorithm. Suppose the robot shown in Figure P3.14 (a) is to go from point R to a target, point T, as shown in Figure P3.14(b). If \(\mathbf{R_{x}}, \mathbf{R_{y}}\), and \(\mathbf{R_{z}}\) are vectors from the robot to each landmar k,X,Y ,Z, respectively, and \(\mathbf{T_{x}}, \mathbf{T_{y}}\), and \(\mathbf{T_{z}}\) are vectors from the target to each landmark, respectively, then heading commands would drive the robot to minimize \(\mathbf{R_{x}-T_{x}}\); \(\mathbf{R_{Y}-T_{Y}}\) ; and \(\mathbf{R_{Z}-T_{Z}}\) simultaneously, since the differences will be zero when the robot arrives at the target (Hong, 1992). If Figure P3.14(c) represents the control system that steers the robot, represent each block—the controller, wheels, and vehicle—in state space.

Admin
Solution

Controller:

The transfer function can be represented as a block diagram as follows:

(first image below)

Writing the differential equation for the first box,

\(\frac{K_{1}}{s+b}\)

and solving for \(\dot{x}_{c} ,\)

\(\dot{x}_{c} =-bx_{c} +K_{1}r_{c}(t)\)

From the second box,

\(y_{c} =\dot{x}_{c} + ax_{c} = -bx_{c} +K_{1}r_{c}(t) + ax_{c}= (a-b)x_{c} + K_{1}r_{c}(t)\)

 

Wheels:

The transfer function can be represented as a block diagram as follows:

(second image below)

Writing the differential equation for the block of the form,

\(\frac{c}{s+c}\)

and solving for\( \dot{x}_{w} \),

\(\dot{x}_{w} = -c x_{w} +cr_{w}(t)\)

The output equation is,

\(y_{w} = x_{w}\)

 

Vehicle:

The transfer function can be represented as a block diagram as follows:

(third image below )

Writing the differential equation for the block,

\(\frac{1}{s}\)

and solving for \(\dot{x}_{v}\)

\(\dot{x}_{v} = r_{v} (t)\)

The output equation is

\(y_{v} = x_{v}\)
